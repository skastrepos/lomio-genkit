# Architecture / 架构

## Workspace / 工作区

```text
apps/
  web/
packages/
  ai/
  cli/
  config/
  core/
  db/
  shared/
```

## Runtime Flow / 运行流程

```text
User input
  -> apps/web page
  -> apps/web API route
  -> @lomio/ai DemoAIProvider
  -> simulated story result
```

用户输入会经过：

```text
用户输入
  -> apps/web 页面
  -> apps/web API Route
  -> @lomio/ai DemoAIProvider
  -> 模拟故事结果
```

## Package Boundaries / 包边界

- `@lomio/ai` owns provider contracts and the Demo Provider.
- `@lomio/core` composes app-level primitives.
- `@lomio/config` reads normalized configuration.
- `@lomio/db` reserves Postgres and Redis boundaries.
- `@lomio/shared` contains tiny shared utilities.
- `@lomio/cli` owns command entry points.

- `@lomio/ai` 负责 Provider 契约和 Demo Provider。
- `@lomio/core` 负责应用级组合能力。
- `@lomio/config` 负责规范化配置读取。
- `@lomio/db` 预留 Postgres 和 Redis 边界。
- `@lomio/shared` 放置小型共享工具。
- `@lomio/cli` 负责命令入口。

## V0.1 Tradeoff / V0.1 取舍

Workspace packages export TypeScript source for local development speed. Build outputs are still generated by each package, and publish-ready package exports can be tightened later.

为了提升本地开发速度，工作区包在 V0.1 中直接导出 TypeScript 源码。每个包仍可生成构建产物，后续发布前可以再收紧 package exports。
